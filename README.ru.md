# Описание 😼

Проект созданный в учебных целях.</br>
Моей задачей было создания рабочего приложения для на Flet потратив ровно столько времени, сколько это нужно было, без учёта красиво дизайна, крутого и мощного стэка и излишних проблем.
Задача была проста, дабы мы могли добавлять задание и удалять.

# Функционал приложения ✅

Приложение умеет:
 - добавлять новые задания;
 - удалять задания;
 - сохранять уже существуют задания в JSON файл.

# Установка и запуск 🛠️

### 1) Настройка окружения.

#### Windows: CMD
```
python -m venv .venv
.venv\Scripts\activate
```

#### Windows: Powershell
```
python -m venv .venv
.\.venv\Scripts\Activate.ps1

При ошибке Execution Policy:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### Linux / macOS
```
Если нет python-venv:
sudo apt install python3-venv

python3 -m venv .venv
source .venv/bin/activate
```

### 2) Установка зависимостей

```
pip install flet pydantic pyinstaller
```
P.s Если вы не хотите в будущем самостоятельно создавать `.exe` файл, рекомендую удалить pyinstaller, дабы не мусорить в проекте.

### 3) Запуск.
```
flet run -m app.main -r -d
```

### 4) Создание exe.
```
flet pack app/main.py --name TodoFletApp --product-name "Todo Flet App" --onedir --distpath release -y
```
P.S. Если вы хорошо понимаете Flet и хотите собирать проект через `flet build windows`, это тоже возможный вариант. В моём случае стабильнее сработал `flet pack`.

# Ошибки ❌

**Ошибка:** Программа/exe-file не запускается или запускается, но вылетает.
**Решение:** Проверьте что папка `_internal` лежит рядом с исполняемой папкой.

**Ошибка:** Файл сохранение не запоминает и/или не загружает задачи.
**Решение:** Не устанавливайте portable-версию в `C:\Program Files`. Перенесите приложение в папку `%LOCALAPPDATA%\Ambre\TodoFletApp` или в любую другую папку, где у программы есть права на запись.

**Ошибка:** Файл задач поврежден.
**Решение:** Удаление файла задач по пути корневой папки, а дальше `data/tasks.json`. Хоть это и редкая ошибка, которая чаше возникает только из-за того что в него залазил сам пользователь, но если вы не хотите капаться в `data/tasks.json` лучше просто удалите его.

**Ошибка:** Антивирус блокирует программу или запуск программы.
**Решение:** Такое тоже бывает, что ваш антивирус может ругается на данную программу, даже если она полностью безопасна. В таком случае советую самостоятельно убедиться что программа не вредоносная, а после добавить в исключение в вашем антивирусе.
